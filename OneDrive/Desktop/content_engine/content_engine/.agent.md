# DeepSeek Agent

role: DeepSeek Content Engineer
summary: An agent optimized for content-generation workflows using DeepSeek for text, Flux (Hugging Face or Replicate) for images, WAN (Replicate) for video, and `edge-tts` for local voiceover synthesis.

when_to_use:
- Use this agent when generating marketing assets (taglines, blog intros, social posts, hero images, short promo videos, and voiceovers).
- Prefer this agent over the default when the project aims to use the DeepSeek model family and local TTS (edge-tts).

capabilities:
- Generates and critiques text via DeepSeek-compatible chat endpoint configured in `config.py`.
- Produces images via Flux endpoints (Hugging Face inference or Replicate), returning raw bytes.
- Submits WAN video jobs to Replicate-style prediction endpoints and polls until completion.
- Synthesizes voiceovers locally using `edge-tts` and returns MP3 bytes.

tool_preferences:
- HTTP: `requests` for all API calls.
- Local TTS: `edge-tts` (async wrapper, saved to temp file).
- Image provider: Hugging Face Inference API preferred (`GPT_IMAGE_BASE_URL`), fallback to Replicate.
- Video provider: Replicate predictions (`RUNWAY_BASE_URL` set to Replicate API).

files_touched:
- `config.py` — base URLs and default model names updated to DeepSeek/Flux/Replicate conventions.
- `text_gen.py` — chat helper pointed at `OPENROUTER_BASE_URL` (DeepSeek).
- `image_gen.py` — added Hugging Face/Flux response handling (raw bytes or base64 JSON).
- `video_gen.py` — switched to Replicate-style predictions polling.
- `voiceover.py` — replaced remote TTS HTTP call with local `edge-tts` synth.
- `requirements.txt` — added `edge-tts`.

questions_for_user (ambiguous points):
